Nearly 6 months after PCSX2 2.4.0 release, we are now nearing the end of 2025.

**Today we are excited to unveil PCSX2 2.6.0!** We're even more excited to showcase the things that we have been working on.

## PCSX2 2.6.0

Now that the dust has settled down from [our biggest single release to date](../../2024/pcsx2-2-release/index.mdx), we decided to step back a bit and give more attention to Quality of Life features and Accessibility! Our contributors have been hard at work further refining the foundation that has been laid out since 2.0.0 to make PCSX2 more accessible to everyone!

</PCSX2PRLink>

TheTechnician27 has been working on adding Korean game titles to most NTSC-K serials which did not previously have them. PCSX2 now has a 99% complete Korean translation on Crowdin (at the time of writing), so this has been a long time coming. We clearly don't expect Korean speakers to speak English to use PCSX2. Yet despite NTSC-K having nearly 700 known serials, the GameDB for a long time has only had Latin versions of the titles, either pulled from the NTSC-U/PAL name or from a Romaji version of the Japanese name.

## Honorable Mentions

<PCSX2PRLink authors="TheTechnician27" prNums="13388">
  ### Add Qt Buddies and Tabstops
</PCSX2PRLink>

This addition helps with screen reading Accessibility and how settings in PCSX2 is presented within PCSX2. [Buddy](https://doc.qt.io/qt-6.8/designer-buddy-mode.html) and [Tabstops](https://doc.qt.io/qt-6.8/designer-tab-order.html) are a very useful feature provided by Qt that allows you to build a more accessible user interface.

#### Buddies

Buddies are a property in Qt which associate a label with an interactive element. This is transparent for most users, but for users with a screen reader, this means you can actually tell what an interactive element does. It's a win for accessibility with minimal maintenance overhead and arguably better self-documentation.

Currently, for example, focusing a dropdown will just say that you've focused a dropdown without saying what it's supposed to be for – rendering much of our UI totally unusable. With a buddy, it will read out the label on top of the element type. As an example, with buddies, the OSD scale option will read approximately as "OSD Scale: Spin Box 100", but without, it will only read "Spin Box 100". There is no way to focus the labels either, so it's unlikely they can be read out without buddies.

<Image src={require("./img/buddy.webp").default} />

#### Tabstops

Have you been in a situation where you have to use a keyboard to navigate your way through PCSX2 and found out that the ordering between options are a complete nonsense? Tabstops is here to help!

Tabstops are another property of Qt which allows an explicit ordering of elements when tabbing through a widget (i.e. focusing on one element then pressing Tab). Without them, the tab order is subject to the physical ordering of the elements in XML, which does not inherently correspond to the row/column order displayed and is highly unreliable and unrealistic to maintain.

<PCSX2PRLink authors="TheLastRar" prNums="13466,12864,13360">
  ### Improved Wayland Support
</PCSX2PRLink>

We have enabled Wayland support by default since PCSX2 2.4.0 release and in this cycle, TheLastRar yet again spearheaded the work to improve Wayland support even further. This time we're cutting out the crufts of technical debts and implement it the way (Qt) intended.

For more technical details, previously the way we render into "surfaces" was to just rendering it to the widget directly. This is not the proper way to do it and has caused some issues especially with Wayland. Instead, Qt recommends using `createWindowContainer()` over rendering into a widget directly. And so that's what we did!
